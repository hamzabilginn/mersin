2. Katmanlı Mimari (Layered Architecture) ve Yapay Zeka

Sistem Monolitik değil, API-First ve Mikroservis Genişlemeye Uygun N-Tier mimaride tasarlanmıştır.

[SUNUM KATMANI] (UI / Frontend) ├─ Web Panel (React) - Tech Office, SC, PM └─ Mobil App (Flutter) - HoM (Offline-First) │ HTTPS / TLS 1.3 (SSL Pinning) [API GATEWAY KATMANI] ├─ Auth & RBAC Middleware (JWT Denetimi) └─ Rate Limiting & ZZZ Cache Middleware │ [İŞ/BUSINESS KATMANI] (.NET Core 8) ├─ Plan Service & Fact Service ├─ AI Agent Service (GPT-4o + Whisper) └─ Sync Service (Offline Idempotency) │ [VERİ KATMANI] (Data Access) ├─ PostgreSQL (Ana Veri + View'ler) └─ Redis (WBS Cache + Queue)

Merkezi İnovasyon: Omnipresent AI

Platform, veri depolamanın ötesine geçerek kullanıcılara asistanlık yapar:

  • HoM İçin (Saha): Sesli komutla WBS doldurma. (Whisper API sesi JSON verisine çevirir).
  • PM İçin (Yönetim): 7/24 Anomali Dedektörü. (Örn: "30CCC lokasyonunda TOW-15 işi 3 gündür %50 verimin altında.")

3. UI Yaklaşımı (Desktop / Mobile)

Kullanıcı deneyimi (UX), personelin şantiye ortamındaki fiziksel kısıtlamalarına göre optimize edilmiştir.

Mobile UI (Head of Master)

  • Teknoloji: Flutter (iOS / Android)
  • Tasarım Felsefesi: Şantiyede eldivenle kullanıma uygun devasa butonlar, yüksek kontrastlı renkler (Güneş altında okunabilirlik).

  • Anlık Bildirim (SignalR): PM ekranında, SC onayladığı an yeni satırlar sayfayı yenilemeden (real-time) belirir.

5. REST API Tasarımı

Tüm endpoint'ler API Gateway üzerinden geçer. JWT Claim tabanlı yetkilendirme (RBAC) ile korunur.

[GET] /api/v1/wbs └─ WBS hiyerarşisi (TOW/STOW/SSTOW) ve ZZZ kodlarını döndürür (Redis Cache). Mobil app sabah online iken çekip lokale kaydeder. [POST] /api/v1/plans └─ Tech Office plan oluşturur. (Body: date, kkk, project, zzz, qty, hom_email) [GET] /api/v1/plans/assignments?date={date} └─ JWT içindeki HoM email'ine göre filtreli görevleri döndürür. [POST] /api/v1/facts └─ Normal (Online) gün sonu veri girişi. [POST] /api/v1/facts/sync └─ OFFLINE'dan gelen bulk (toplu) veri. Dizi (Array) kabul eder. Header: Idempotency-Key (Çift kaydı önler). [PATCH] /api/v1/approvals/{fact_id} └─ SC veya PM onayı. Body: { action: "APPROVE"|"REJECT", comment: "..." } [GET] /api/v1/reports/daily?kkk=30DDD └─ Power BI'ın çektiği otomatik hesaplanmış (Efficiency dahil) rapor VIEW'i.

7. Offline Senaryosu (Connectivity Strategy)

Nükleer santral sahası gibi kalın betonarme ve yeraltı yapılarında sinyal kaybı kaçınılmazdır. Çözüm:

Senkronizasyon Yaşam Döngüsü

  1. Sabah (Ofiste): Mobil cihaz WBS listesini ve atanan görevleri ana sunucudan alır, Hive Local DB'ye kaydeder.
  2. Saha (Sinyal Yok): HoM veriyi girer (Fact Qty, Overtime vs.). Veri lokale yazılır. Durum: Pending Sync.
  3. Akşam (Sinyal Geldi): Cihaz WiFi/4G yakaladığı an /api/v1/facts/sync tetiklenir. Tüm paket asenkron gönderilir.

Çakışma Koruması (Idempotency)

Uygulama internete bağlanırken anlık kopmalar yaşanırsa aynı veri sunucuya 2 kez gidebilir.

Bunu önlemek için cihaz offline iken bir UUID (local_id) üretir. Sunucu bu ID'yi Redis üzerinde 24 saat saklar. Eğer aynı ID tekrar gelirse, sunucu işlemi görmezden gelir (200 OK döner ama DB'ye çift yazmaz).

8. Güvenlik Yaklaşımı

  • RBAC (Role-Based Access Control) ve Hiyerarşi: Authentication (JWT Claims) ile organizasyonel hiyerarşi tam yalıtılır:
    • HoM: Sadece atandığı Lokasyonu (KKK) yönetir.
    • Tech Office (Submitted By): Sadece sorumlu olduğu bölgenin (crew region) planını oluşturur.
    • Site Chief: Sadece kendi bölgesinin (crew region) onaylarını yapar. (Yatay yetki sızıntısı engellenir).
    • Project Manager: Tüm projeyi kapsayan en üst düzey onay merciidir.
  • Immutable Audit Trail (Değiştirilemez Log): approvals tablosuna kimin ne zaman onay verdiği yazılır, bu tabloya SQL UPDATE/DELETE işlemi asla uygulanmaz. Soft Delete kullanılır.
  • Mobil Güvenlik: Tüm ağ (Transport) katmanı TLS 1.3'tür. SSL Certificate Pinning kullanılarak Ortadaki Adam (MitM) saldırıları önlenir.

9. Teknoloji Seçimi ve Gerekçeleri (MVP & Ölçeklenebilirlik)

KatmanTercihAlternatifGerekçe
Frontend React + TS (Web)
Flutter (Mobil)		 Vue.js
React Native		 React'in sanal grid performansı binlerce ZZZ kodu render etmek için mükemmeldir. Mobil için Flutter, cihazın lokal DB'siyle %100 offline çalışma garantisi sunar.
Backend .NET Core 8 (C#) Node.js / Laravel Kurumsal ERP sistemlerine (SAP vb.) entegrasyon gücü ve güçlü tip güvenliği (Type Safety). Çok kanallı asenkron işlemlerde yüksek performans.
Database PostgreSQL SQL Server Açık kaynak lisanssızdır. Power BI Native DirectQuery ve JSONB desteği ile hiyerarşik ZZZ sorgularında mükemmel hız sağlar.
Cache Redis Memcached Sadece cache değil, binlerce kullanıcının akşam offline'dan online'a geçtiğinde yığılan verilerini (Queue) kilitlemeden yönetebilmek için In-Memory çözüm.
Workflow MassTransit + RabbitMQ Hangfire Onay süreçlerindeki (HoM -> SC -> PM) durum değişikliklerini olay güdümlü (Event-Driven) yönetmek ve bildirimleri anlık iletmek için mikroservis mimarisine en uygun yapı.
Authentication OAuth 2.0 + JWT Session Cookie Stateless (durumsuz) olması sayesinde mobil uygulamanın offline senaryolarda token ile lokalde oturumu korumasına izin verir. RBAC kolayca entegre edilir.
Deployment Docker (On-Premise) Cloud (AWS) Nükleer tesis güvenlik standartları gereği dış buluta (Cloud) kapalı, sadece yerel ağda (Intranet) çalışacak izole konteyner mimarisi.
