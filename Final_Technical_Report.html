Workforce Execution Platform

Final Technical Report & Architecture Comparison

Hazırlayan: Mustafa Hazar Bilgin

Proje Özeti (Documentation):
Sunumu gerçekleştirilen MVP projesindeki "Dokümantasyon / Raporlama" ekranlarında da kanıtlandığı üzere; sistemin temel amacı T-1 (Planlanan) ve T0 (Gerçekleşen) adam-gün / miktar (Planned vs Fact) verilerini ZZZ Kodu (WBS Hiyerarşisi) üzerinden şeffaf bir şekilde kıyaslayabilmektir. Mimaride Offline-First yaklaşımı ve "-1 İş Başlamadı" kuralı başarıyla uygulanmıştır.

1. Mimari Teknoloji Karşılaştırması

Canlı demoda sunulan PHP tabanlı MVP, uç senaryoları (Offline PWA, AI Agent, Onay akışları) hızlıca sahada kanıtlamak (Proof of Concept) için geliştirilmiştir. Yeni.pdf sunumunda yer alan Hedef Kurumsal Mimari (Target Enterprise Architecture) ise sistemin Nükleer/İnşaat regülasyonlarında canlıya çıkacağı nihai N-Tier mimaridir.

Katman MVP Teknolojisi (Prototip) Kurumsal Mimari (Target - yeni.pdf) Tercih Gerekçesi (Hedef Mimari)
Backend / API PHP & Laravel 11 .NET Core 8 (C#) Yüksek asenkron performans, kurumsal ERP/SAP uyumluluğu ve strongly-typed güvenli yapı.
Database SQLite PostgreSQL WBS (ZZZ Kodu) ağaç sorgularında Recursive CTE gücü ve düşük lisans maliyeti.
Frontend Vue.js 3 React.js / Next.js Binlerce ZZZ kodunu DataGrid üzerinde sanallaştırma (Virtualization) ile hızlı render edebilme.
Mobile App PWA (Browser) Flutter Sahada hem iOS hem Android için native donanım erişimi ve %100 Offline Storage desteği.
Message Broker Senkron REST RabbitMQ / MassTransit Tech Office, HoM ve SC onay akışlarında (Workflow) sistemin kilitlenmesini önleme.
Deployment Local Server Docker (On-Premise) Güvenlik regülasyonları gereği verinin iç ağda (On-Prem) izole konteynerlerde tutulması.

Hem MVP'nin hem de hedef mimarinin kalbini oluşturan ilişkisel veritabanı şeması aşağıdaki gibidir:

users Roller (RBAC)
id (PK) integer
role enum (tech_office, hom, sc, pm)
name string
email string
tasks T-1 ve T0 Birleşik MVP Tablosu

zzz_code string

planned_qty decimal (T-1 Plan)
fact_qty decimal (T0 Gerçek)
comment text (-1 Mazereti)
status enum (assigned, pending_sc, approved)
tech_office_id (FK) integer
hom_id (FK) integer
sc_id (FK) integer
pm_id (FK) integer
ai_messages AI Action Agent Logları
id (PK) integer
user_id (FK) integer
sender enum (self, ai)
text text
